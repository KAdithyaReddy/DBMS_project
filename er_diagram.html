Logical Database Design - ER Diagram





Entities:

Strong Entities: State, Energy, Power Plant, User, Login

Weak Entities: Hydro, Bio Power, Solar, Wind

Attributes:

State: StName (PK), Year, Solar, Wind, Bio_Power, Small_Hydro, Large_Hydro, Potential, Total

Energy: Source (PK), Cost_of_Generation, Capital_Cost, Allotted, Commissioned, Cancelled, Balance (Derived), Generation_in_Million_Units (Derived)

Power_Plant: EType, EGen, Address (PK), Area, Cost

User: UserID (PK), Username, UserMobile, User_Email, User_Addr

Login: LoginId (PK), LoginUsername, Password

Hydro: Area, Energy_Generated, Cost, Type, State (FK), No._of_Dams

Bio_Power: Area, Energy_Generated, Cost, Type, State (FK), No_ of_Plants

Solar: Area, Energy_Generated, Cost, Type, State (FK), No_of_Farms

Wind: Area, Energy_Generated, Cost, Type, State (FK), No._of_Farms

Relationships:

State Consumes Energy: Many to Many (Binary Relationship)

User Has Login: One to One (Binary Relationship)

Power Plants Generate Energy: Many to Many (Binary Relationship)

Energy Has Forms of Energy: One to One (Ternary Relationship)

Schema:

State (StName, Year, Solar, Wind, Bio_Power, Small_Hydro, Large_Hydro, Potential, Total )

Energy (Source, Cost_of_Generation, Capital_Cost, Allotted, Commissioned, Cancelled)

Power_Plant (EType, EGen, Address, Area, Cost)

User (UserID, Username, UserMobile, User_Email, User_Addr)

Login (LoginId, LoginUsername, Password)

Hydro (Area, Energy_Generated, Cost, Type, State, No_of_Dams)

Bio_Power (Area, Energy_Generated, Cost, Type, State, No_of_Plants)

Solar (Area, Energy_Generated, Cost, Type, State, No_of_Farms)

Wind (Area, Energy_Generated, Cost, Type, State, No_of_Farms)
